# 产品需求文档 (PRD)

## 一、产品定位

本产品按云服务器部署标准开发（无需实际部署，验证演示可本地/局域网启动），核心是网页端群聊工具（复刻抖音聊天功能、融合微信群聊便捷操作），集成AI聊天助手（API调用模式，不本地部署，兼容多厂商API，通过密钥接入），支持群聊与好友一对一对话双模式。

**产品用途：**

- 个人学习实践
- C++后端开发工程师求职简历呈现

**开发特点：**

- 开发适配云服务器部署，验证演示时可本地/局域网启动满足小用户量测试
- 整体轻量、稳定、易实现
- 重点完善AI API接入、好友添加逻辑、登录注册安全校验及AI回复规范

---

## 二、核心功能列表

### 2.1 基础聊天功能

包含群聊与好友对话双模式，按云服务器部署架构开发，验证演示时本地/局域网可正常使用，重点完善好友添加逻辑、登录注册安全校验：

#### 群聊

- 支持多人群聊创建
- 成员管理（邀请、移除、禁言）
- 消息发送（文字、图片、表情）
- 群公告编辑与推送
- 群聊创建后将生成专属群聊ID，用户可通过群聊ID向群主发起加群申请
- 群主参照好友请求审核逻辑，对加群申请进行"同意"或"拒绝"操作，审核结果即时反馈给申请人
- 支持创建群聊时上传自定义群聊头像，后续可随时修改
- 支持所有群成员查看群成员列表（显示成员头像、用户名、用户ID）
- 支持群主及群管理员修改群名，修改后所有群成员同步可见

#### 好友对话

- 支持一对一聊天
- 可添加、删除好友、设置备注
- 发送文字/图片/表情
- 支持消息已读/未读、撤回
- 聊天时可查看好友头像及用户名
- 好友头像、用户名修改后同步更新

#### 好友添加逻辑（参考微信，通过用户ID添加）

1. 发起添加请求（输入对方用户ID，可附验证消息）
2. 对方接收请求（查看请求方用户名、用户ID及验证消息）
3. 对方选择"同意"或"拒绝"
4. 同意后双方互为好友，可发起对话
5. 拒绝后对方无法再次发送请求（可设置1小时冷却期），同时记录好友请求状态便于追溯

#### 动态验证码

- 仅用于辅助密码登录，不可替代密码
- 验证码为系统随机生成的6位字符（数字+字母组合，区分大小写）
- 以图像形式展示（可设置刷新按钮），用户需准确输入图像中的字符完成验证
- 验证码有效时间5分钟，1分钟内不可重复获取，使用一次后失效
- 后端记录验证码生成时间、有效期、使用状态，避免重复验证

#### 账号管理

- 登录与注册在同一网页页面，支持自由切换模式，操作便捷
- 登录时采用用户ID登录，需输入用户ID、密码，并完成动态验证码（图像形式）辅助验证，三者均验证通过方可登录
- 支持用户上传自定义头像，后续可随时修改头像
- 支持用户后续修改用户名（用户名可重复，修改后即时生效）

#### 请求管理

**好友请求：**

- 接收好友添加请求
- 可查看请求方用户名、用户ID、头像及验证消息
- 明确显示"同意""拒绝"两个选项
- 操作后即时反馈结果

**加群请求：**

- 群主接收用户通过群聊ID发起的加群申请
- 可查看申请人用户名、用户ID、头像及申请消息
- 参照好友请求审核逻辑，选择"同意"或"拒绝"
- 同意则申请人加入群聊并同步群聊头像、群名等信息
- 拒绝则告知申请人并记录审核状态

### 2.2 AI聊天助手功能（API调用，不本地部署）

#### AI接入方式

- 采用多厂商API兼容设计，不进行本地部署
- 仅需在代码中修改API和key即可完成不同厂商的接入
- 具体接入厂商在代码编写阶段确定，无需用户操作
- 后端封装统一API调用接口

#### AI调用方式

支持两种调用方式，适配不同使用场景：

**按键调用：**

- 在聊天对话框（群聊、一对一对话）底部设置AI助手调用按键
- 点击按键即可召唤AI助手，直接输入提问内容，无需额外操作

**@召唤：**

- 在聊天输入框中输入"@+AI助手自定义昵称"
- 跟随输入提问内容，发送后即可触发AI助手响应

#### AI助手自定义修改

- 在聊天框设置选项中实现
- 支持用户对AI助手进行个性化定义
- 可修改AI助手昵称、语气（活泼、严谨、幽默）、响应优先级
- 无需配置API相关参数，API及密钥已在代码编写阶段确定
- 修改后即时生效并应用于后续AI响应

### 2.3 AI回复要求及优化思考（速度与质量兼顾）

核心要求：AI回复需简洁、精准，避免冗杂，优先切分为多个短对话发送，同时兼顾响应速度与回复质量

#### 回复规范

**简洁性：**

- 单条AI回复不超过50字
- 直击问题核心，不添加无关冗余描述
- 避免大段文字堆砌

**分条发送：**

- 若问题需多方面解答（如包含2个及以上核心要点）
- 拆分多个短对话逐次发送
- 每条仅包含1-2个核心点
- 提升阅读体验，避免一次性发送大段内容导致阅读繁琐

**精准性：**

- 过滤AI回复中的无关、错误内容
- 确保回复与用户提问高度相关，不偏离主题
- 若出现无效回复，触发二次API调用（最多2次）
- 若仍无法获取有效回复，提示用户"当前AI回复异常，请稍后再试"

#### 速度与质量兼顾的思考

**响应速度优化：**

- 将AI API调用超时时间设置为1-2秒，避免用户长时间等待
- 利用Redis缓存高频提问及对应AI回复
- 对于重复提问，直接从缓存中获取回复，无需重复调用API，大幅提升响应速度

**回复质量保障：**

- 后端新增AI回复校验逻辑
- 对接收到的AI回复进行关键词匹配、语义校验
- 过滤无效、冗余内容，确保回复精准
- 支持用户在聊天框设置选项中调整AI回复优先级（质量优先/速度优先），平衡速度与质量

**异常处理：**

- 针对API调用失败、网络异常等情况，设置友好提示
- 记录错误日志，便于后续排查问题
- 若多次调用失败，自动切换备用AI厂商（需提前配置备用厂商API参数），保障AI助手可用性

## 三、用户使用流程

### 3.1 基础聊天与账号操作流程

#### 1. 验证演示启动

- 发起者启动本地后端、数据库服务
- 配置AI API基础参数（预留厂商配置位）
- 打开前端页面或好友通过局域网访问主机IP+端口
- 进入登录/注册同一页面

#### 2. 登录/注册

- 选择注册（设置用户名、用户ID、符合要求的密码）
- 或选择登录（输入用户ID、密码，完成图像形式的动态验证码辅助验证）
- 完成操作后进入聊天主界面

#### 3. 好友添加流程

1. 发起添加 → 输入对方用户ID，可附验证消息 → 发送添加请求
2. 对方接收请求，查看请求方用户名、用户ID及验证消息
3. 对方选择"同意"/"拒绝"
4. 同意则添加成功，可发起一对一对话
5. 拒绝则添加失败，需等待冷却期后再次尝试

#### 4. 群聊/好友对话

- 添加好友后可发起一对一聊天
- 聊天时可查看好友头像及用户名，好友头像、用户名修改后即时同步显示
- 可创建群聊（创建时可上传群聊头像，生成专属群聊ID）
- 或通过群聊ID向群主发起加群申请，待群主审核同意后加入群聊
- 群内可查看所有群成员列表（显示成员头像、用户名、用户ID）
- 群主及管理员可修改群名，修改后所有群成员同步可见
- 用户可随时修改个人头像、用户名，修改后好友列表及群聊中对应信息同步更新
- 完成基础聊天及个性化设置操作

---

### 3.2 AI助手使用流程

#### 5. AI设置

- 在聊天框的设置选项中操作
- 用户可对AI助手进行修改和定义（如修改昵称、语气、响应优先级）
- AI的API及密钥已在代码编写阶段确定，无需用户配置
- 仅需在代码中修改API和key即可完成不同厂商的接入

#### 6. AI调用

- 在群聊或一对一对话界面
- 可通过点击聊天框底部AI助手按键
- 或输入"@+AI助手昵称"，发送提问内容

#### 7. AI响应

- 后端接收请求后，调用配置的AI API
- 对返回的回复进行校验、拆分（若需）
- 逐次发送至聊天界面，完成咨询

#### 8. 个性化调整

- 用户可随时通过聊天框的设置选项，修改AI助手相关配置（昵称、语气、响应优先级），适配不同使用场景
- 同时可在个人中心修改自身头像、用户名
- 群主及管理员可在群聊设置中修改群聊头像、群名
- 所有修改均即时同步生效
